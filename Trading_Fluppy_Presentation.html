🏗️ Архитектура системы ENGINE ↔ VISUAL

Принцип двухфайловой архитектуры

Система разделена на два индикатора, работающих как единый механизм. ENGINE выполняет все вычисления и передаёт результаты в VISUAL через невидимые plot-каналы.

ENGINE indicator() → 51× plot() каналов → input.source() маппинг → VISUAL indicator() → Дашборд + Hints
⚡ Важно: TradingView strategy() не экспортирует plot() для input.source(). Поэтому ENGINE реализован как indicator() с симулированным трейдингом (SIM_INITIAL_CAPITAL = 1000.0).

Версионирование и подключение

VISUAL автоматически определяет подключение к ENGINE по значению первого канала:

EXPECTED_ENG_VERSION = 278.0
_auto_engine = math.round(eng_version) == 278.0

Три режима подключения:

  • Auto (по умолчанию) — автодетекция по Version каналу
  • Force ON — принудительное использование ENGINE данных
  • Force OFF — standalone режим с fallback-расчётами
⚠️ Лимит 64 plot-слота: TradingView ограничивает 64 output slots (plot, plotshape, bgcolor, fill и др.). ENGINE использует ~51 из 64. Добавление новых каналов требует оптимизации существующих.

Порядок подключения в TradingView

  1. Добавить ENGINE индикатор на график
  2. Добавить VISUAL индикатор поверх ENGINE
  3. TradingView автоматически маппит каналы по порядку
  4. Проверить статус в дашборде: 🟢 = подключено

OHLCV данные → Auto-Tuner (TF+Asset) → Индикаторы (RSI, ATR, EMA, VWAP) → Пивоты + BOS/CHoCH → Кластеризация → Зоны → Скоринг (6 компонент + NN) → Entry Filters → Sim Position + Risk → 51× plot() Export

🏛️ Типы зон и алгоритмы обнаружения

Система обнаруживает 6 основных типов зон, каждый с уникальным алгоритмом детекции, валидации и скоринга.

📦

Order Blocks (OB)

Зоны поддержки/сопротивления — основной тип. Кластерный анализ свечей вблизи пивотов. f_cluster() сканирует назад от пивота, находит свечи одного направления, группирует по допуску ATR.

Бычий OB: Кластер красных свечей перед разворотом вверх.

Медвежий OB: Кластер зелёных свечей перед разворотом вниз.

🔄

Rejection Zones (RZ)

Зоны отклонения — мощное отклонение цены от уровня через длинные тени. Wick-to-body ratio на пивотах.

Бонусы: RSI дивергенция (+15), Hidden Rejection (+12), Quadratic Wick (до +20).

📐

Fair Value Gaps (FVG)

Ценовые разрывы — 4 подтипа: Classic FVG, Vol Imbalance, Marubozu, Open-Open. Валидация: Volume, Z-Score, размер в ATR.

💔

Breaker Blocks (BB)

Пробитые OB — конвертация с обратной полярностью. Скор: 0.8 x old_score + momentum. Cascade limit: 2.

🚀

Propulsion Blocks

OB + FVG на одной свече — экстремальный институциональный интерес. Бонус: +20.

🛡️

Mitigation Blocks

BB прошедший ретест — подтверждение смены полярности. Бонус: +20.

Жизненный цикл зоны

Пивот → f_cluster() → Скоринг → Бонусы → BEST/STRONG/NORMAL → Aging → Invalidate/BB

Алгоритм f_cluster() — детальная схема

// Вход: swing_idx (индекс пивота), is_bull (направление)
f_cluster(swing_idx, is_bull) =>
    expected_color = is_bull ? "red" : "green"
    for i = 1 to ob_len
        if candle[swing_idx - i].color == expected_color
            if abs(candle.close - cluster_avg) <= p_cluster_tol x ATR
                cluster.add(candle)
    zone_top = max(cluster.highs), zone_bot = min(cluster.lows)
    score = f_compute_zone_score(z, rvol, trend, sweep, eff, pa) + bonuses + nn
    return [zone_top, zone_bot, min(score, 150.0), ...]

🎯 Система скоринга зон

6-компонентная модель оценки качества

Каждая зона оценивается по 6 независимым компонентам. Веса нормализуются к сумме = 100%. Финальный скор ограничен диапазоном [0, 150].

КомпонентВес (дефолт)АлгоритмДиапазон
Z-Score (Аномальность)25% Сигмоидная нормализация: sigmoid((z_score - z_min) / (z_max - z_min)). Измеряет насколько свеча статистически аномальна по размеру тела. 0-100
Volume (RVOL)15% Логарифмическая нормализация: log(RVOL / mom_rvol) x 50. Относительный объём показывает институциональную активность. 0-100
Trend Alignment15% Бинарная проверка: зона совпадает с направлением HTF/ITF тренда = полный вес, иначе = 0. 0 или weight
Sweep / Liquidity Grab15% Обнаружение sweep ликвидности вблизи зоны: 0.6x (минимальный sweep) или 1.0x (глубокий sweep). 0 - weight
Efficiency (FER)15% Fractal Efficiency Ratio: net_move / total_move за последние eff_len баров. Высокий FER = чистое направленное движение. 0-100
PA Patterns (Бонус)15% Свечные паттерны: Break Engulfing, Sweep Volume, Pin Bars. Каждый подтверждённый паттерн добавляет к компоненту. 0 - weight

Confluence Score (конфлюэнция)

Функция f_calc_confluence_score() проверяет совпадение зоны с ключевыми уровнями. Каждое совпадение: +5 очков (максимум 30).

Fib 38.2% Fib 50% Fib 61.8% Fib 78.6% VWAP Round Level Prev Day H Prev Day L Prev Day C

Допуск совпадения: 0.2 x ATR от уровня. Confluence score добавляется к итоговому скору зоны.

Стохастическая модель прогнозирования

Биварная модель (время до ретеста + объём при ретесте) с экспоненциальным затуханием весов:

// Формула весов наблюдений
w = exp(-2 / n * age)

// Seasonality для крипто (24ч цикл)
seasonal = 1 + 0.1 * sin(2*PI*hour/24) + 0.05 * sin(2*PI*day/5)

// Confidence scaling
prob = 50 + (raw - 50) * sqrt(n / max_n)

// CDF Hastings approximation (precision 1.5e-7)

⚡ Логика входа и выхода

Три режима входа

Touch (Limit) — По умолчанию

Цена касается края зоны. Требуется подтверждение pin-bar (min 2) и reverse spike (RVOL > 3.0) или pattern combo. Самый точный, но может пропустить быстрые движения.

Рекомендуется для: M5-M30, все фьючерсные пары

Candle Close

Свеча полностью закрывается внутри зоны. Требуется pin-bar + impulse. Более надёжный, но вход чуть хуже (дальше от edge).

Рекомендуется для: H1-H4, BTC/USDT и ETH/USDT

ChoCh (Confirm)

Требуется BOS (Break of Structure) + закрытие внутри зоны. Самый консервативный, наименьший win rate но наилучший R:R.

Рекомендуется для: D1+, volatile пары (ZEC/USDT)

🛡️ Риск-менеджмент

Система управления рисками

📐

Kelly Criterion

Оптимальный размер позиции: f = (WR x AvgW - (1-WR) x AvgL) / AvgW. Clamp [0, 1]. Рекомендуется использовать Half-Kelly (f/2) для снижения variance.

📊

Sharpe Ratio

Annualized: mean(returns) / std(returns) x sqrt(252). Хороший: > 1.0, Отличный: > 2.0. Показывает доходность на единицу риска.

📉

Sortino Ratio

Как Sharpe, но учитывает только downside deviation. Более релевантен для торговли: mean(returns) / downside_std x sqrt(252).

🎲

Monte Carlo Bootstrap

500 путей x 50 шагов с block bootstrap (block 3-5). 95th percentile maximum drawdown. Показывает worst-case DD с 95% уверенностью.
